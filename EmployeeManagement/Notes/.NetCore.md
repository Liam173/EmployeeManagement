## Data Flow in .NET Core:

# Controller:
Basic request validation.
Eg. "Do you have a boarding pass?"

```
[HttpPost]
public IActionResult Create(Employee employee)
{
    _employeeService.Create(employee);

    return Ok();
}
```

# Service:
Business logic and data processing.
Eg. "Are you allowed on the flight?"

```
public void Create(Employee employee)
{
    if (employee.Age < 18)
    {
        throw new InvalidOperationException(
            "Employee must be at least 18.");
    }

    _repository.Add(employee);
}
```

# Repository:
It doesn't care why they're flying.
Eg. Seats the passengers.

```
_repository.Add(employee);
```

# Database:
Data integrity.
Eg. The seat physically only fits one passenger.

Data integrity rules can be enforced at the database level to ensure that the data remains consistent and valid. For example, you can enforce constraints such as:

```
Age > 0

Salary >= 0

NOT NULL

Unique Email
```

## Logging Levels in .NET Core:

# Trace: Extremely detailed
eg. Every SQL parameter.

# Debug: Developer information
eg. Variable values

# Information: Normal application flow
eg. Employee created

# Warning: Something unexpected but reversable
eg. Employee not found

# Error: Operation failed
eg. Exception thrown

# Critical: Application cannot continue
eg. Database unavailable
